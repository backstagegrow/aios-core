Quick Start Guide

1. Custom Typography

To adapt this system for a new client, generate a fresh Google Fonts link and replace it in the <head>. Then, update the tailwind config object:

tailwind.config = { theme: { fontFamily: { sans: ['NewFont', 'sans-serif'], display: ['ClientDisplay', 'serif'], } } }

2. System Colors

The core palette is mapped to the stone color object in Tailwind. To completely re-theme the page for a brand, simply overwrite the stone keys or replace bg-stone-900 with the client's primary accent.

colors: { stone: { // Rename or override 900: '#BrandPrimary', ... }, 'surface-light': '#BrandBg', }

3. Media Assets

This base uses placeholder Unsplash IDs and Iconify icons (e.g. solar:icon-name). For production implementations, search the HTML for <img> tags and replace src with local client assets.

<!-- Example Icon Swap --> <iconify-icon icon="solar:medic-bold"> <!-- Image Swap --> <img src="client-hero.webp">
Copy this exact HTML file into any new project folder to serve as the unified component base.
