## Filosofia Financeira

Thiago Finch tem uma visão clara sobre como tratar o dinheiro no negócio digital:

> "Faturamento é vaidade. Lucro é sanidade. Fluxo de caixa é realidade."

Princípios:
- **Margem primeiro**: de nada adianta faturar muito se margem for baixa
- **Reinvestimento inteligente**: parte do lucro vai para crescer o negócio
- **Controle de custos**: custos fixos baixos = mais margem
- **LTV > CAC**: o valor do cliente precisa superar o custo de aquisição

---

## Modelo de Unidade Econômica (Unit Economics)

### Fórmula base
```
Receita por Cliente - CAC - Custo do Produto - Custo Operacional = Lucro por Cliente
```

### Exemplo numérico (produto info a R$997)
```
Receita: R$997
(-) Comissão plataforma: R$50 (5%)
(-) Imposto (Simples): R$60 (6%)
(-) CAC (custo de ads): R$200
(-) Custo do produto: R$30
(-) Suporte: R$20

= LUCRO LÍQUIDO: R$637 por cliente
= Margem: ~64%
```

## Métricas Financeiras Essenciais

| Métrica | Descrição | Meta |
|---------|-----------|------|
| **CAC** | Custo de Adquirir um Cliente | O mais baixo possível |
| **LTV** | Valor Total do Cliente no Tempo | 3x a 5x o CAC |
| **ROAS** | Retorno sobre Gasto em Anúncio | > 3x |
| **Margem Bruta** | % de receita após custo direto | > 70% para info |
| **Margem Líquida** | % de receita após todos os custos | > 30% |
| **MRR** | Receita Mensal Recorrente | Crescimento consistente |
| **Ticket Médio** | Valor médio por venda | Aumentar com upsells |

## Estratégia de Preços

### Precificação por Valor (Value-Based Pricing)
```
Resultado entregue ao cliente = R$10.000
→ Cobrar de 10% a 30% do resultado = R$1.000 - R$3.000
```

### Ancoragem de Preço
```
"De R$1.997 por apenas R$997"
→ Percepção de desconto aumenta conversão
```

### Escala de Tickets
| Produto | Ticket | Margem Típica |
|---------|--------|---------------|
| Ebook/Mini curso | R$27-R$97 | 90%+ |
| Curso completo | R$297-R$997 | 75-85% |
| Mentoria grupo | R$2.000-R$10.000 | 60-75% |
| Mentoria individual | R$10.000-R$50.000 | 70-85% |

## Maximização de Receita por Cliente

### Order Bump
```
No checkout, oferta adicional de baixo valor
Exemplo: Curso R$497 + Bônus exclusivo por R$47
→ Aumenta ticket médio em 15-30%
```

### Upsell
```
Após compra, oferta de produto superior
Exemplo: Curso vendido → Mentoria em grupo ofertada
→ Aumenta LTV significativamente
```

### Downsell
```
Para quem não comprou, oferta inferior
Exemplo: Não comprou mentoria → oferta do curso basic
→ Recupera vendas perdidas
```

## Estrutura de Custos Típica

### Negócio Digital Saudável
```
RECEITA: 100%
  (-) Anúncios (tráfego): 25-35%
  (-) Plataforma (Hotmart, etc.): 4-8%
  (-) Impostos: 6-15%
  (-) Suporte/Atendimento: 3-5%
  (-) Ferramentas (SaaS): 2-3%
  (-) Produção/Conteúdo: 2-5%

= MARGEM LÍQUIDA: 30-55%
```

## Fluxo de Caixa e Reinvestimento

### Regra de Reinvestimento (modelo ensinado)
```
Receita Líquida → 70% reinvestir → 30% retirar
```

Breakdown do reinvestimento:
- 40%: tráfego pago
- 20%: time e estrutura
- 10%: produto e conteúdo

### Projeção de Escala
```
Mês 1: R$30k faturado → reinveste R$20k
Mês 2: R$60k faturado → reinveste R$40k
Mês 3: R$120k faturado → reinveste R$80k
```

## Impostos e Estrutura Jurídica

Recomendações frequentes para negócios digitais:

- **PJ desde o início**: pessoa jurídica paga menos imposto
- **Simples Nacional**: para faturamento até R$4,8M/ano
- **Lucro Presumido**: pode ser mais vantajoso em faturamentos maiores
- **Holding**: proteção patrimonial e otimização fiscal

## Patrimônio e Segurança Financeira

Mentalidade patrimonial ensinada:

> "Transforme renda em patrimônio. Só renda é frágil."

Estratégia:
```
Negócio gera renda
→ Renda gera poupança
→ Poupança investe em ativos (imóveis, ações, FIIs)
→ Ativos geram renda passiva
→ Liberdade financeira
```
