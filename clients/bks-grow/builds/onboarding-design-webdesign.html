Etapa 1

Briefing e Acessos

Tudo que precisa ser coletado antes de abrir o Figma ou escrever uma linha de código.

Materiais do Cliente

  • Logo em todos os formatos (SVG, PNG fundo transparente, vetor)
  • Manual da marca (se existir) ou referências visuais
  • Paleta de cores oficial (hex/rgb)
  • Fontes usadas pela marca
  • Banco de imagens e fotos do produto/serviço
  • Vídeos brutos para edição
  • Textos, copy e informações das páginas

Acessos Necessários

  • Acesso ao Drive / repositório de arquivos do cliente
  • Domínio do cliente — painel de DNS (para apontar Cloudflare Pages)
  • Contas de redes sociais (para publicação de posts)
💡 Criar pasta padrão no Drive assim que receber os materiais. Estrutura: Logo / Imagens / Vídeos / Textos / Exports
Etapa 2

Design System no Figma

Base de todo o trabalho visual — configurada uma vez, usada em tudo.

Setup do Arquivo Figma

  • Duplicar o template de novo cliente como base
  • Renomear o arquivo com o nome do cliente
  • Configurar as variáveis de cor (tokens) com a paleta do cliente
  • Configurar tipografia — heading, body e mono
  • Ajustar espaçamentos e border-radius da marca
  • Importar e organizar o logo em todas as variações

Componentes a Montar

🎨
Tokens de Cor e Tipografia
Primária, secundária, acento, texto, muted, fundo
🔲
Componentes Base
Botões, cards, badges, inputs, ícones padronizados
📐
Templates de Posts
Stories, feed quadrado, carrossel, reels (thumbnail)
🌐
Templates de Web
Hero, seções, cards, footer — alinhados ao HTML que será construído
Etapa 3

Identidade e Direção Visual

Definir o look & feel antes de produzir qualquer peça.

Checklist de Identidade

  • Validar paleta de cores (primária, secundária, acento)
  • Confirmar tipografia em todos os pesos necessários
  • Definir estilo visual: minimalista, bold, clean, premium, vibrante
  • Levantar referências visuais com o cliente (moodboard)
  • Definir padrão de uso do logo (fundos claros/escuros)
  • Garantir coerência entre digital e impresso (se houver)

Estilos Visuais Comuns

⬛
Dark Premium
Fundo escuro, acentos vibrantes, tipografia forte
⬜
Light Clean
Fundo claro, espaço em branco, elegância
🎯
Bold Direct
Cores fortes, fontes pesadas, impacto imediato
✨
Soft Modern
Gradientes suaves, arredondados, contemporâneo
⚠️ Coerência é obrigatória. O visual dos posts, da página e dos vídeos precisa parecer da mesma marca.
Etapa 4

Produção de Peças

Materiais visuais para redes sociais, campanhas e comunicação da marca.

📱
Feed (Posts Estáticos)
Formato 1:1 ou 4:5. Prova social, frases, posicionamento. Export PNG/JPG.
🎠
Carrossel
Múltiplos slides com progressão de conteúdo. Export sequencial nomeado.
📸
Stories
Formato 9:16. CTA, bastidores, promoções. Versões com e sem texto.
🎬
Thumbnail de Reels
Cover do vídeo alinhado com o estilo visual do feed.
🖼️
Banners e Anúncios
Múltiplos formatos para Meta e Google. Hierarquia visual clara.
📦
Organização de Entrega
Exportar com nomenclatura padrão. Pasta por tipo. Nunca enviar arquivo Figma bruto.

Etapa 5

Produção de Vídeo

Vídeo faz parte do escopo de design — mesmo fluxo, mesma identidade.

Checklist de Vídeo

  • Receber vídeos brutos do cliente no Drive
  • Alinhar roteiro ou script antes da edição
  • Aplicar identidade visual: cores, fontes, logo e vinheta
  • Editar com ritmo adequado ao formato (Reels ≠ institucional)
  • Adicionar legenda / captions (obrigatório para Reels)
  • Exportar nos formatos corretos por plataforma
  • Nomear arquivos com padrão antes de entregar

Formatos de Entrega

🎞️
Reels / TikTok
9:16 vertical · até 90s · MP4 · legenda inclusa
🏢
Vídeo Institucional
16:9 horizontal · qualidade alta · MP4 H.264
📣
Vídeo para Anúncios
Múltiplos formatos · primeiros 3s com gancho visual
✂️
Cortes e Variações
Versão longa + versão 15s + versão 6s para testes A/B
Etapa 6

Web — Build da Página

Páginas desenvolvidas in-house em HTML/CSS — sem dependência de plataforma terceira.

Checklist de Desenvolvimento

  • Briefing da página — objetivo, seções e copy definidos
  • Montar wireframe no Figma antes de codar
  • Desenvolver em HTML + CSS seguindo o Design System do cliente
  • Garantir responsividade — mobile primeiro
  • Testar velocidade e performance (Lighthouse)
  • Revisar SEO básico: title, meta, OG tags
  • Testar em diferentes navegadores e dispositivos

Estrutura de Arquivos

# Estrutura padrão do projeto cliente-slug/ ├── index.html ├── assets/ │ ├── css/ │ │ └── style.css │ ├── js/ │ │ └── main.js │ └── images/ │ └── (imagens otimizadas) └── README.md
💡 Nomear o repositório como slug-do-cliente. Ex: bks-grow, house-gt. Facilita o deploy e o apontamento de domínio.
Etapa 7

Web — Deploy e Domínio

Do código local ao ar — pipeline padrão via Cloudflare Pages.

01
Repositório
Código finalizado no repositório do projeto
→
02
Cloudflare Pages
Conectar repositório ao Cloudflare Pages para deploy automático
→
03
URL de Preview
Cloudflare gera URL temporária para aprovação do cliente
→
04
DNS do Cliente
Adicionar CNAME no painel DNS apontando para Cloudflare Pages
→
05
✅ No Ar
Página publicada no domínio do cliente com HTTPS automático

Configuração DNS — Padrão

# No painel de DNS do cliente Tipo: CNAME Nome: www (ou subdomínio) Valor: [slug].pages.dev # Para domínio raiz (apex) Tipo: CNAME Nome: @ Valor: [slug].pages.dev

Checklist de Deploy

  • Testar URL de preview antes de apontar domínio
  • Confirmar que SSL/HTTPS está ativo
  • Verificar redirect www → domínio principal
  • Testar formulários e links em produção
  • Confirmar Pixel e GA4 disparando corretamente
  • Registrar a URL final no Drive do cliente
Etapa 8

Aprovação e Entrega

Nada vai para o cliente sem revisão interna. Nada vai ao ar sem aprovação do cliente.

Fluxo de Aprovação

1
Interno
Revisão de Qualidade
Checar identidade visual, coerência, erros de texto e performance
2
Cliente
Envio para Aprovação
Link de preview (web) ou PDF/arquivo (peças). Prazo de feedback: 24h-48h
3
Ajustes
Máximo 2 Rodadas
Revisar feedbacks e aplicar alterações. Mais de 2 rodadas indica briefing incompleto
✅
Entrega
Publicação / Deploy
Arquivos no Drive do cliente + página no ar + confirmação por escrito

Checklist de Entrega Final

  • Todos os arquivos exportados com nomenclatura padrão
  • Pasta no Drive do cliente atualizada e organizada
  • Página no ar, testada e com domínio apontado
  • Pixel e rastreamento confirmados em produção
  • Aprovação do cliente registrada por escrito
  • Resumo da entrega enviado ao cliente
⚠️ Sem aprovação escrita, não existe entrega concluída. Sempre confirmar por mensagem ou e-mail.
